## Implementations

### 1. metadataBase + Open Graph + Twitter Cards
- `src/app/layout.tsx` — metadataBase, OG locale fr_FR, Twitter summary_large_image
- Image OG generee dynamiquement via `src/app/opengraph-image.tsx` (Edge runtime)

### 2. Sitemap dynamique
- `src/app/sitemap.ts` — pages statiques + fetch dynamique tous les slugs restaurants
- Accessible via `/sitemap.xml`

### 3. robots.txt
- `src/app/robots.ts`
- Allow : `/`, `/r/*`, `/m/*`, pages legales, signup
- Disallow : `/dashboard/*`, `/s/*`, `/kds/*`, `/api/*`

### 4. JSON-LD Structured Data
- `src/shared/components/JsonLd.tsx` — composant reutilisable
- Organization schema + SoftwareApplication schema dans root layout
- FAQPage schema dans la section FAQ de la landing

### 5. Favicon + PWA Manifest
- `public/favicon.svg` — SVG avec gradient bleu
- `public/manifest.json` — PWA standalone, theme bleu

### 6. Google Analytics
- `src/shared/components/GoogleAnalytics.tsx`
- Env var `NEXT_PUBLIC_GA_ID` (a configurer dans Vercel)
- anonymize_ip active, strategy afterInteractive

### 7. Canonical URLs
- `/r/[slug]` et `/m/[slug]` ont des canonical URLs dans generateMetadata

### 8. Protection pages privees
- Dashboard layout : `robots: { index: false, follow: false }`
- robots.txt bloque `/dashboard/*`, `/s/*`, `/kds/*`

### 9. FAQ Schema
- JSON-LD FAQPage injecte dans FAQSection avec les 6 questions existantes

### 10. Lazy Loading
- Landing page : sections below-the-fold en `dynamic(() => import(...))`
- HeroSection chargee immediatement pour le LCP
